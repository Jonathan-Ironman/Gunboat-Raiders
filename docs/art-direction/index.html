Design System
Harbour Dawn
Premium naval combat — fresh, not childish. High legibility under fast action.

Design Principles

Premium naval strategy — dark, deep, cinematic. Feels like a real game, not a flash toy.
High legibility always — every text element must be readable at a glance under fast action. No exceptions.
Warm gold accent system — amber/gold carries energy and status. Teal carries health and safety. Red carries danger. Nothing else carries semantic weight.
Fresh, not childish — Baloo 2 adds warmth without being a kids game font. Clean geometric UI language.
Restraint in decoration — one gradient per surface at most. Glow effects only on interactive or active states.

Depth System
Elevation & Surfaces
Four surface levels create depth hierarchy. Never skip levels — each layer must be visibly distinct from the one below.
Level 0 — Background
#071120 · bg-deep
Darkest — behind everything. Game canvas sits here.
Level 1 — Page Background
#0f1e36 · bg
Base page/HUD background. Primary canvas for content.
Level 2 — Surface
#152844 · surface
Cards, panels, HUD containers. Box-shadow: --shadow-sm.
Level 3 — Elevated Surface
#1c3556 · surface-el
Hover states, nested panels, focused inputs. Box-shadow: --shadow-md.
Level 4 — Modal / Overlay
rgba(7,17,32,.9) + backdrop-filter: blur(8px)
Pause menu, dialogs. Box-shadow: --shadow-lg.

Animation
Motion Principles
Fast and purposeful. Animations communicate state, not style. Nothing decorative animates unless it carries information.

What Animates — and What Does Not

SHOULD ANIMATE
Health bar width — 0.2s ease-out on change
Damage numbers — float up + fade 1.2s
Buttons — translateY on hover/press (instant)
Modal — scale + fade on enter/exit
Toast — slide from right on enter (0.3s)
Hit marker — flash 0.15s then fade
Critical health bar — pulse loop animation
Ammo slot fill — slot fills left-to-right on reload
Score — countup tally on wave complete
Wave announce — large number scale-in 0.5s spring
MUST NOT ANIMATE
HUD layout position — never shifts during play
Text labels — no wiggle, bounce, or typewriter
Background elements — no parallax or drift
Static borders and cards — no breathing/pulsing
Crosshair idle state — must be perfectly still
Score/wave counter during combat — update instantly
Nav / minimap — no transitions on updates
Disabled states — no animation, just opacity .35
Visual Language
Imagery & Iconography

Icon Style

Style: Outlined, single weight
Stroke weight: 1.5–2px at 24px
Corner radius: 2px (rounded, not sharp)
Fill: Solid fill only for active/selected states
Sizes: 16px (caption) · 20px (menu) · 24px (HUD) · 32px (large)
Color: Inherit from text context — never hardcoded unless semantic (red=danger, gold=active, teal=ready)

Texture / Pattern

HUD panels: Flat — no texture
Modal overlays: backdrop-filter blur(8px) only
Gradient use: Max 1 gradient per surface. Directional only (no radial bursts in UI)
Glow effects: box-shadow only, never CSS filter:blur on UI elements (performance)
Scanlines / noise: Not used — keeps feel clean and legible

3D Asset Integration

Models: Kenney CC0 — clean, low-poly toon-adjacent
Lighting key: Gold-tinted directional (dawn angle)
Water: Gerstner waves, custom shader — navy-to-teal gradient, gold specular highlights
HUD/3D boundary: HUD is an HTML overlay — never attempt to blend 3D elements into the HUD layer
